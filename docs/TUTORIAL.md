# Digital Power 多格式翻译工作台 · Step by Step 教程

本教程按"部署 → 配置 → 三大翻译场景 → 进阶技巧"的顺序，带你完整走一遍工作台的全部功能。

---

## 第 0 步：部署

### 方式 A：Vercel 一键部署（推荐）

1. 打开 [vercel.com](https://vercel.com)，用 GitHub 账号登录。
2. 点击 **Add New → Project**，导入仓库 `Shawn-Skywolfer/MultipleLanguageTranslator`。
3. 配置全部留空即可：
   - Framework Preset：`Other`
   - Build Command / Output Directory / Install Command：留空
4. 点击 **Deploy**。Vercel 会自动把根目录静态文件发布为前端页面，把 `api/*.js` 识别为 Serverless Functions。
5. （可选）如需全网持久的 PV 统计，在 Vercel 项目设置中添加环境变量 `KV_REST_API_URL` 和 `KV_REST_API_TOKEN`（Upstash Redis）。

### 方式 B：本地调试

```bash
npx vercel dev
```

访问终端输出的本地地址。注意：直接双击 `index.html` 或用纯静态服务器打开时，`/api/*` 代理不可用，模型调用会失败。

## 第 1 步：模型配置

页面顶部的"模型配置"卡片是一切功能的前提。

1. **Provider 预设**：可直接点选内置预设（AIHubMix、智谱 BigModel、Kimi Code），Base URL 和模型列表会自动填充；也可以手动填写任何 OpenAI 兼容接口。
2. **Base URL**：形如 `https://aihubmix.com/v1` 的 OpenAI 兼容地址。
3. **API Key**：你的密钥。可点击"保存配置"记住在本机浏览器（localStorage），"忘记配置"可清除。
4. **模型 ID**：从下拉列表选择，或点击"刷新模型列表"从接口拉取；支持输入关键字过滤。
5. 点击 **连通测试**：成功会显示模型返回的确认信息。测试不通过时请先检查 Base URL 是否带 `/v1`、Key 是否有效。

> 所有模型调用都经站内 `/api/chat-completions` 代理转发，浏览器不会直接暴露请求给第三方页面。

## 第 2 步：共享翻译规则与目标语言

这部分配置对 CSV、PDF、PPTX 三种翻译全部生效。

1. **标准翻译库（可选）**：上传 CSV，支持两种格式——
   - 窄表：`source / target_lang / translation` 三列（列名也兼容 `source_text`、`原文`、`language`、`标准译文` 等）；
   - 宽表：`source` + `German` / `French` / `Chinese` 等语言列。
   命中标准译文时有两种模式：**直接使用**（不调用模型，最快且零成本）或 **仍调用完整 DSL 评审**（让模型参考标准译文再精修）。
2. **术语保护**：填写不允许被翻译的术语（每行一个），如品牌名、产品型号；也可以上传术语 CSV（优先读取 `term` / `protected_term` / `术语` / `词条` 列）。翻译完成后会自动校验术语是否在译文中保留，缺失会标 Warning。
3. **自定义规则**：自由文本，如"语气正式""数字保留原格式"等，会注入翻译提示词。
4. **目标语言**：默认提供 18 种欧洲语言，支持全选 / 清空 / 一键选择欧洲主要语言，也可输入自定义语言添加。每个目标语言都会单独产出一份输出件。

## 第 3 步：CSV 批量翻译

适合翻译产品文案、营销物料等表格化内容。

1. 进入"CSV 批量翻译"页，上传一个或多个 CSV 文件（可拖拽）。
2. 设置 **原文列**：填写列名（如 `text`、`原文`）。
3. （可选）设置 **国家列** 或填写 **全局国家**：用于让译文贴合目标市场表达习惯。
4. 选择 **源语言**、**工作流模式**：
   - 完整 DSL：`初译 → 建议 → 改译` 三轮调用，质量最高；
   - 快速模式：单轮直译，速度最快。
5. 设置 **并发数**（1–4）与 **失败重试次数**（0–3）。
6. 选择输出格式：
   - **宽表**：原表追加 `生成结果_German` 等语言列，适合回填系统；
   - **长表**：每行 `原行 + target_lang + translation + error`，适合审校流转。
7. 点击 **开始批量翻译**，进度条实时显示；完成后在结果表格中按语言下载。

## 第 4 步：PDF 翻译（资料快速阅读场景）

适合阅读分析师报告等图表密集的资料型 PDF。

1. 进入"文档翻译"页，上传 PDF（可多个、可拖拽）。
2. 选择源语言（影响 OCR 语言包：中文用 `chi_sim+eng`）、工作流模式、并发与重试。
3. 勾选输出件（可多选）：
   - **双语 Markdown**：按"页 → 文本块"组织的原文/译文对照，适合检索与二次加工；
   - **双语 DOCX**：每页一张"原文 | 译文 | 备注"三列表格，适合审校批注；
   - **图文对照 HTML**（推荐用于阅读）：左侧为原文页面高清截图——图表、版面 100% 原貌；右侧为按阅读顺序排列的译文块；**点击译文块，左侧页面上对应原文区域会高亮**；每块可展开查看原文；支持打印。
4. 点击 **开始文档翻译**。
   - 文字型 PDF 直接抽取文本块（保留坐标）；
   - 整页文本不足 24 字符时自动回退到浏览器端 Tesseract OCR（首次加载语言包较慢，进度会实时显示）。
5. 完成后按"文件 × 语言 × 输出类型"逐项下载。

> 当前限制：位图图表内部的文字（截图粘贴的图）暂不做 OCR 翻译，图表以原样呈现；矢量文字图表的文字会作为文本块参与翻译。

## 第 5 步：PPTX 翻译（版面保真场景）

适合需要保留幻灯片原版面的对外材料。

1. 同样在"文档翻译"页上传 PPTX。
2. 点击开始后，输出一个翻译版 PPTX：**原始页全部保留，每页后新增一页对应译文页**（10 页原稿输出 20 页）。
3. 译文页的布局保真保证：
   - **字号 100% 一致**：逐 run 保留原始字号；原文本框带自动缩放（`fontScale`）时，会把缩放固化进字号后锁定，避免 PowerPoint 因译文变长二次缩小；
   - **位置 100% 一致**：文本框矩形、对齐、锚点全部继承原页，原文居中的译文与其水平垂直同心；
   - **译文不换行**：长度差异沿原对齐方向延伸，不因换行改变行数和垂直位置。
4. 译文文本框以黄色底色高亮，便于审校时快速定位已翻译内容。
5. 翻译过程带行数质检：发现行数缩减、内容合并等异常会自动逐行重译。

## 进阶技巧与常见问题

- **并发与重试怎么选？** 模型接口稳定时并发 3–4 + 重试 1；接口限流明显时并发 1–2 + 重试 2–3。
- **标准翻译库何时用"直接使用"？** 已有人工审定的译文资产时。统计面板会显示库命中次数与未命中语言分布。
- **某些文本块翻译失败？** 输出件中会以 `[ERROR]` 标注并保留原文位置，重跑时只需关注失败项。
- **超大文件**：百页级 PDF / PPTX 在浏览器端处理时间明显增加；PDF 勾选图文对照 HTML 时会逐页渲染截图，文件体积也会增大，可按需取消勾选。
- **界面语言**：右上角可随时切换中文 / English，选择会记住在本机。
