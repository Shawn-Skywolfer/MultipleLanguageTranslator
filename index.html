内置 DSL 工作流结构

本工具按你上传的 Dify DSL 重建前端工作流，并修正了两个明显变量问题：初译 system prompt 中把 source_text 改为 source_lang；最终改译中把 {target_lang} 改为实际目标语言。

说明与限制

文件夹模式依赖浏览器 File System Access API，建议使用最新版 Edge / Chrome。其他浏览器可以用多文件模式并手动下载结果。
完整 DSL 模式每行每个目标语言会调用 3 次模型。比如 100 行 × 5 个语言 = 1500 次模型调用。大批量建议先用 2-3 行测试。
API Key 会出现在浏览器发给 Provider 的请求头中。此单页工具没有后端，不会把 Key 发给除你配置的 Base URL 之外的地址。
